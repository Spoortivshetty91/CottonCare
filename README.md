# 🌿 CottonCare – AI Based Cotton Crop Disease Detection

CottonCare is an AI-powered mobile application designed to help farmers detect cotton crop diseases quickly and accurately using image processing and deep learning techniques. The system uses a Convolutional Neural Network (CNN) model to analyze cotton leaf images and predict diseases such as Bacterial Blight, Curl Virus, Fusarium Wilt, and Healthy crops.

---

## 📌 Description

Cotton is one of the most important crops in agriculture, but it is highly vulnerable to various diseases that reduce crop quality and yield. Identifying these diseases manually requires expert knowledge, which is not always accessible to farmers.

To solve this problem, CottonCare provides a smart and user-friendly solution that allows farmers to capture or upload images of cotton leaves. The system analyzes the image using a trained CNN model (MobileNet) and provides instant disease prediction along with remedies and preventive measures.

## 🚀 Features

- 📷 **Leaf Image Detection**
  - Capture or upload cotton leaf images for disease detection

- 🤖 **AI-Based Prediction**
  - Uses CNN (MobileNet) model for accurate classification

- 🌱 **Disease Classification**
  - Bacterial Blight  
  - Fusarium Wilt  
  - Curl Virus  
  - Healthy Leaf  

- ⚡ **Offline Functionality**
  - Works without internet using TensorFlow Lite model

- 💡 **Remedy Suggestions**
  - Provides treatment and preventive measures after detection

- 📚 **Disease Library**
  - Detailed information about diseases, symptoms, and causes

- 🧠 **Symptom Checker**
  - Identify diseases manually based on visible symptoms

- 📊 **Scan History**
  - Stores previous scan results for tracking crop health

## ⚙️ How It Works

1. User captures or uploads a cotton leaf image  
2. Image is preprocessed (resize, normalization)  
3. CNN model analyzes leaf patterns  
4. System predicts disease category  
5. Displays:
   - Disease name  
   - Confidence level  
   - Remedies & prevention  

This workflow ensures fast and accurate detection directly on the device. :contentReference[oaicite:1]{index=1}
